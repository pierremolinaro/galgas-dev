GALGAS ChangeLog


1.3.0

  • fixed bug on 'allkeys reader' of map objects;
  • added 'object ++ ;' incrementation instruction on @uint and @sint objects;
  • added 'object -- ;' decrementation instruction on @uint and @sint objects;
  • added +, -, *, /, % diadic operators on @uint objects, that return an @uint object;
  • added +, -, *, /, % diadic operators on @sint objects, that return an @sint object;
  • added - monoadic operator on @sint object, that returns an @sint object;
  • added 'uint' reader on @sint object, that returns an @uint object with the same value;
  • added 'string' reader on @sint object, that returns an @string object with the string value of the receiver;
  • added 'min' constructor on @sint object, that returns an @sint object with value equal to -217483648;
  • added 'max' constructor on @sint object, that returns an @sint object with value equal to 217483647;
  • added '-=' operator on @stringset object, that removes a string from a string set;
  • added 'setWithString' constructor on @stringset type, that creates a new string set containing a string;
  • suppressed 'empty' constructor on @stringset type: use 'emptySet' constructor instead;
  • suppressed 'empty' constructor on list types: use 'emptyList' constructor instead;
  • suppressed 'empty' constructor on map types: use 'emptyMap' constructor instead;

    1.2.5beta2

    • This is an unstable release
    • new keywords: 'root', 'entity', 'metamodel';
    • suppressed keyword: 'start';
    • grammar component change: now 'root' keyword is used instead of 'start' keyword for name the start symbol;
    • fixed a lexique component bug: 'send' instruction did not check the terminal symbol to be declared;
    • fixed a grammar component bug: output HTML file was not generated when the source file was written in command line without any path;
    • fixed a bug when linking on MSYS/MinGW: now strip is called, instead of mingw32-strip;

    1.2.4

    • added --create option for creating ready-to-use project.
    • galgas tutorial compiles now correctly.
    • added allKeys and unreadKeys readers to maps.
    • added union, intersection and difference operators to @stringset;
    • compilation by makefile under Win32 now needs MSYS.

    1.2.3

    • corrected LL(1) parser bug: on syntax error, 1.2.2 and earlier galgas and generated applications and tools that use LL(1) parser can display an incorrect list of expected terminal symbols, or even crash.
    • galgas compiles now correctly under MinGW.
