GALGAS ChangeLog


1.3.2

  • ouput effective arguments now support polymorphic assignment;
  • an empty lexical analyzer now raises lexical errors on every character until the end of source file (on previous versions, it runs indefinitly);

1.3.1

  • Version 1.3.0 has been deleted; --create-project option did not work;
  • on @bool type: added ^ (exclusive or) diadic operator on @bool objects, that returns an @uint object;
  • on @uint type: added 'object ++ ;' incrementation instruction on @uint objects;
  • on @uint type: added 'object -- ;' decrementation instruction on @uint objects;
  • on @uint type: added + (addition) , - (subtraction), * (multiplication) , / (division), % (modulo), & (bit wide and), | (bit wide or), ^ (bit wide exclusive or) diadic operators on @uint objects, that return an @uint object;
  • on @sint type: added 'object ++ ;' incrementation instruction on @sint objects;
  • on @sint type: added 'object -- ;' decrementation instruction on @sint objects;
  • on @sint type: added +, -, *, /, % diadic operators on @sint objects, that return an @sint object;
  • on @sint type: added - monoadic operator on @sint object, that returns an @sint object;
  • on @sint type: added 'uint' reader on @sint object, that returns an @uint object with the same value;
  • on @sint type: added 'min' constructor on @sint object, that returns an @sint object with value equal to -217483648;
  • on @sint type: added 'max' constructor on @sint object, that returns an @sint object with value equal to 217483647;
  • on @sint type: added 'string' reader on @sint object, that returns an @string object with the string value of the receiver;
  • on @stringset type: suppressed 'empty' constructor: use 'emptySet' constructor instead;
  • on @stringset type: added 'setWithString' constructor, that creates a new string set containing a string;
  • on @stringset type: added '-=' operator, that removes a string from a string set;
  • on user defined map type: fixed bug on 'allkeys accessor' of map objects;
  • on user defined map type: suppressed 'empty' constructor: use 'emptyMap' constructor instead;
  • on user defined map type: search methods on map are now readers (instead of modifiers): calls them [map_expression search_method ...] instead of [!?map_expression search_method ...];
  • on user defined map type: added 'mapWithMapToOverride' constructor, that creates an empty map that overrides an other map;
  • on user defined map type: added 'overridenMap' reader, that returns the overriden map;
  • on user defined list type: suppressed 'empty' constructor on list types: use 'emptyList' constructor instead;
  • --create-project Command Line Option replaces --create Command Line Option: now generates Win32 Code::Blocks project;
  • New Keyword:mapindex;

1.2.5beta2

  • This is an unstable release
  • new keywords: 'root', 'entity', 'metamodel';
  • suppressed keyword: 'start';
  • grammar component change: now 'root' keyword is used instead of 'start' keyword for name the start symbol;
  • fixed a lexique component bug: 'send' instruction did not check the terminal symbol to be declared;
  • fixed a grammar component bug: output HTML file was not generated when the source file was written in command line without any path;
  • fixed a bug when linking on MSYS/MinGW: now strip is called, instead of mingw32-strip;

1.2.4

  • added --create option for creating ready-to-use project.
  • galgas tutorial compiles now correctly.
  • added allKeys and unreadKeys readers to maps.
  • added union, intersection and difference operators to @stringset;
  • compilation by makefile under Win32 now needs MSYS.

1.2.3

  • corrected LL(1) parser bug: on syntax error, 1.2.2 and earlier galgas and generated applications and tools that use LL(1) parser can display an incorrect list of expected terminal symbols, or even crash.
  • galgas compiles now correctly under MinGW.

1.2.2

  • First avalaible release on this site.
