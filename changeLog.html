GALGAS ChangeLog

1.5.5

  • Some error messages in french are now translated in english [18];

1.5.4

  • Fixed a bug on map remove modifier;
  • Better error message on undeclared map search method call;

1.5.3

  • New @uint constructor: sequenceNumber that returns an unique value;
  • New command line option: -S or --shadow, that warns when a variable declaration shadows a previously declared variable;
  • New map modifier: remove modifier; a remove modifier performs entry removing;
  • New short cut in modifier call instruction;
  • New short cut in method call instruction;
  • New short cut in foreach instruction;

1.5.2

  • New @string modifier: setCapacity that sets the receiver capacity;
  • New @string reader: capacity that returns the receiver current capacity;
  • New @string reader: uppercaseString that returns an uppercased representation of the receiver;
  • New @uint reader: hexString that returns hexadecimal representation of the receiver;
  • New @uint64 reader: hexString that returns hexadecimal representation of the receiver;
  • Added the .= operator that appends an @string to an @string;
  • Fixed a bug on @string stringByReplacingStringByString reader;

1.5.1

  • Fixed a bug [14] on using an option in a semantic source file;
  • Syntax change: in a class declaration, the reader key word is replaced by the method key word [see 1.5.1 release notes];
  • New @string constructor: stringWithCurrentDateTime that returns a string containing the current date time;
  • From now, all @string are available for @lstring objects;
  • From now, concatenation operator '.' concats an @string object with an @lstring object, returning an @string object;
  • Fixed a bug [17] on count reader of @stringset type: it was badly declared in GGS_stringset C++ class (LIBPM revision 265);

1.5.0

  • Now all types accept == and != operators [13];
  • Removed unused feature in cocoa gui component: see [1.5.0 release notes];
  • makefile for Mac OS X has been modified so that compilation using GCC 3.3 can be performed without any change;
  • Fixed a bug in Cocoa GALGAS on version number checking;

1.4.9

  • WARNING: INCOMPATIBILITY ISSUE; see [1.4.9 release notes];
  • New command line option: -W or --Werror, that treats warnings as errors (similar to -Werror gcc command line option);
  • Improved list generated code: GALGAS generates smaller code size, using a common super class: AC_galgas_list;
  • New @string reader: characterAtIndex that returns the character at index given by argument value;
  • New @string reader: length that returns the number of characters;
  • New @string constructor: stringWithCurrentDirectory that returns the current directory;
  • New @uint constructor: errorCount that returns the number of errors;
  • New @uint constructor: warningCount that returns the number of warnings;
  • GALGAS generated code now contains checkings for detecting GALGAS version mismatch against libpm library version;

1.4.8

  • Syntax change: '??' is no longer used in foreach, extract and match instructions; a warning is raised for every deprecated syntax;
  • GALGAS tools and GALGAS generated tools no longer retain unuseful objects: all allocated objects are now released;
  • New @string constructors: stringWithSourceFilePath, stringWithContentsOfFile, stringWithMainFirstArgument;
  • New @string method: writeToFile;
  • Entries are now correctly searched in overriden maps;
  • Shadow keys are now correctly detected in overridden map [12];

1.4.7

  • Added two lexical actions (thanks to Mikaël Briday [9]): escape sequences in strings were not recognized if they occur at the end of the string;
  • Added many features on list types (see 1.4.7 release notes);
  • Added a new log instruction;
  • Reader description now displays variable value in an indented form;

1.4.6

  • Fixed a bug [8]: enterOctDigitIntoUInt and enterOctDigitIntoUInt64;
  • In GALGAS generated XCode projects, the -Wshadow flag is not set any more [7];
  • Added a feature that enables getting key word spelling in syntax;
  • Added in Cocoa Galgas two menu items: "Block Comment", for commenting out several lines in one time, and "Block Uncomment", the reverse operation;
  • Added a switch instruction enumerated values;

1.4.5

  • Fixed a bug [4] on >> operator;
  • Fixed a bug [5] on >= operator;
  • Fixed a bug [6] on "GALGAS Cocoa (ppc)" target of Galgas XCode project: an absolute path was used instead of a relative one;
  • Fixed a bug that prevents cocoa Galgas to open an UTF-8 encoded text file;

1.4.4

  • Fixed a bug [1] that crashes Cocoa GALGAS and Cocoa GALGAS generated applications when opening an empty source text file;
  • Added a feature [2] in map insert method declaration for raising an error when an insertion shadows a previously declared object;
  • Fixed a bug [3] that prevents GALGAS compiler to recognize mapWithMapToOverride as a valid map constructor;
  • Added automatic update for Cocoa GALGAS application (see "Update" tab in Preferences windows);

1.4.3

  • Added two new lexical actions (thanks to Mikaël Briday): enterBinDigitIntoUInt and enterBinDigitIntoUInt64 that enable building an 32-bit and 64-bit unsigned value from binary string;
  • In Cocoa GALGAS Applications (and Cocoa GALGAS generated applications), the ruler thickness is now adjusted to display 5 digit line numbers.
  • From now, GALGAS compilers (and GALGAS generated tools) run silently without any message on success (of course, warnings and errors are still displayed);
  • From now, GALGAS compilers (and GALGAS generated tools) have a new command line option --verbose (shortly -v) that enables message displaying, as for previous releases.

1.4.2

  • Added --libpm command line option for defining libpm path on project creation
  • Fixed a bug in Cocoa application that prevents vertical text view ruler to be updated on text font change;
  • Fixed a bug in loop instruction generation.

1.4.1

  • New 64-bit integer types @uint64 and @sint64;
  • New 64-bit integer types with location @luint64 and @lsint64;
  • value accessor of types with location is deprecated; use instead bool accessor for @lbool type, ... (see release notes);
  • Types with location now accept modifiers;
  • Project creation options now create Mac OS X Xcode project;
  • GALGAS and libpm can be now considered as 64-bit clean.

1.4.0

  • First attempt to provide galgas and libpm code 64-bit clean: now, makefiles build both 32-bit and 64-bit tools;
  • Consequently, the makefile_i386linux_on_macosx directory is now named makefile_x86linux_on_macosx;
  • makefiles are now invoked with --warn-undefined-variables command line option (strangely, compilation under MSYS on Windows warns that '^' variable is not defined)
  • Ouput effective arguments now support polymorphic assignment;
  • An empty lexical analyzer now raises lexical errors on every character until the end of source file (on previous versions, it runs indefinitly);
  • Many limitations on class handling have been relaxed (see Release notes);

1.3.1

  • Version 1.3.0 has been deleted; --create-project option did not work;
  • on @bool type: added ^ (exclusive or) diadic operator;
  • on @uint type: added 'object ++ ;' incrementation instruction, 'object -- ;' decrementation instruction, + (addition) , - (subtraction), * (multiplication) , / (division), % (modulo), & (bit wide and), | (bit wide or), ^ (bit wide exclusive or) diadic operators;
  • on @sint type: added 'object ++ ;' incrementation instruction, 'object -- ;' decrementation instruction, +, -, *, /, % diadic operators, - monoadic operator, 'uint' and 'string' readers, 'min' and 'max' constructors;
  • on @stringset type: suppressed 'empty' constructor (use 'emptySet' constructor instead), added 'setWithString' constructor, added '-=' operator (that removes a string from a string set);
  • on user defined map type: fixed bug on 'allkeys accessor', suppressed 'empty' constructor (use 'emptyMap' constructor instead), search methods on map are now readers (instead of modifiers);
  • on user defined list type: suppressed 'empty' constructor (use 'emptyList' constructor instead)
  • --create-project Command Line Option replaces --create Command Line Option: now generates Win32 Code::Blocks project;

1.2.5beta2

  • This is an unstable release
  • new keywords: 'root', 'entity', 'metamodel';
  • suppressed keyword: 'start';
  • grammar component change: now 'root' keyword is used instead of 'start' keyword for name the start symbol;
  • fixed a lexique component bug: 'send' instruction now checks the terminal symbol is actually declared;
  • fixed a grammar component bug: output HTML file was not generated when the source file was written in command line without any path;
  • fixed a bug when linking on MSYS/MinGW: now strip is called, instead of mingw32-strip;

1.2.4

  • added --create option for creating ready-to-use project.
  • galgas tutorial compiles now correctly.
  • added allKeys and unreadKeys readers to maps.
  • added union, intersection and difference operators to @stringset;
  • compilation by makefile under Win32 now needs MSYS.

1.2.3

  • corrected LL(1) parser bug: on syntax error, 1.2.2 and earlier galgas and generated applications and tools that use LL(1) parser can display an incorrect list of expected terminal symbols, or even crash.
  • galgas compiles now correctly under MinGW.

1.2.2

  • First avalaible release on this site.
