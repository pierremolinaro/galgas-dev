GALGAS ChangeLog

2.3.4

  • Bug Fix: many improvments and bug fixes on Cocoa GALGAS;
  • Bug Fix: declaration of several arraies with same index count was generation incorrect C++ code;

2.3.3

  • Bug Fix: "double free" crash on Linux;

2.3.2

  • Bug Fix: GALGAS array was hanging on C++ compilation;

2.3.1

  • Bug Fix: Cocoa GALGAS graphic user interface indexing;
  • Bug Fix: @string popen returns correct values;
  • Useless files in generated libpm directory are from now automatically deleted;
  • New array type;
  • New range type;
  • New @double exp reader: that computes the base-e exponential of the receiver's value;
  • New @double logn reader: that computes computes the value of the natural logarithm of the receiver's value;
  • New @double log10 reader: that computes computes the value of the logarithm of the receiver's value to base 10;
  • New @double log2 reader: that computes computes the value of the logarithm of the receiver's value to base 2;

2.3.0

  • New Cocoa GALGAS graphic user interface;
  • New @string popen reader (does not work on Windows);

2.2.7

  • Bug fix: incorrect generation of += operator within modifiers;

2.2.6

  • New feature: hasKeyAtLevel reader for maps;

2.2.5

  • New feature: setter and nogetter features in a list declaration;
  • Bug Fix: @function isFunctionDefined constructor was returning not valuated object;

2.2.4

  • Bug fix: modifying a list element attribute could affect data integrity;
  • Bug fix: using indexing with gsyntax rules labels did generate incorect code;
  • Removed feature: readers declared in an enum declaration;

2.2.3

  • Bug fix: enumerated object variables are from now detected as unavailable within 'before', 'between' and 'after' blocks of foreach instruction;
  • Obsolete feature: readers declared in an enum declaration are obsolete (use a category reader instead);

2.2.2

  • New feature: 'parse' non terminal labels are implicitly generated;
  • New feature: improved map 'search' error messages;

2.2.1

  • Bug fix: incorrect code generation for if instruction, when test expression is not an @bool;
  • New feature: category readers, methods and modifiers have been implemented for every type;
  • New feature: abstract category modifiers, overriding category modifiers overriding abstract category modifiers have been implemented for class types;
  • New feature: error messages are colored in red in Unix console;

2.2.0

  • New feature: external LIBPM is no longer used; GALGAS compiler now embbeds LIBPM, and generates it a the GALGAS_OUPUT directory of every project;
  • Bug fix: incorrect code generation for category readers;
  • LIBPM Bug fix: allocation of zero length array;

2.1.8

  • Bug fix: in GALGAS 2.0.0 to 2.1.7, collection comparisons were always false;

2.1.7

  • Bug fix: some structured instructions were accepting silently erroneous variable definitions;
  • New target option: makefile_x86_64_linux_on_macosx for generating a makefile for cross compiling on Mac OS X for 64-bit Linux;
  • GALGAS now uses the new cross compiler on Mac OS X for 32-bit Linux;
  • GALGAS now uses the new cross compiler on Mac OS X for 64-bit Linux;

2.1.6

  • for Unix: from now, one makefile, that builds a 32-bit executable on a 32-bit system, and a 64-bit executable on a 64-bit system;
  • New command line option: --build=string that invokes a makefile for building a specific target;
  • New option for the scanner for performing word or full token selection in Cocoa editor;

2.1.5

  • indexing is now invoked with cmd key (instead of the alt key);
  • issue messages are colored (use --no-color command line option for disabling this feature);

2.1.4

  • Fixed bug: self and selfcopy invocation in category reader;

2.1.3

  • Fixed bug on non detection of some undefined types;
  • Fixed bug : non constant function formal parameters were generated as constant formal parameters;
  • Fixed bug : semantic import circularity is now detected;
  • map log now displays overriden maps;

2.1.2

  • New feature: indexing in Cocoa GALGAS;
  • Fixed struct access bug: checking was incorrect, this has been fixed;
  • Declaring a method within a class is from now deprecated; a warning is issued for every method declaration;
  • Use the --no-deprecated-warning for preventing warnings on deprecated constructs;
  • Functions, routines and filewrappers can nom be declared local to the current file, by prefixing their declaration with the local keyword;

2.1.1

  • Fixed cast expression bug: generated code was incorrect, this has been fixed;
  • Fixed variable state expression bug: checking was incorrect, this has been fixed;
  • New type: graph;

2.1.0

  • Fixed bug of --version command line option;
  • Enhanced target settings in project;
  • Reader and method categories are available;

2.0.1

  • New internal BDD package, optimized for 64-bit executables;

2.0.0

  • First release of GALGAS 2: use --new-code-generation command line option for enabling GALGAS 2 code generation;
  • Change: when enumerating a map the lkey variable is used for naming the current key (instead of key) ;
  • Change: map keyList reader is obsolete: now use allKeyList reader;
