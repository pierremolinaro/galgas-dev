\definitionSectionType{@location}

An \nomType{@location} object value is a location in a source file. Objects of this type are useful for pointing out an error or a warning location.

\subsection{The \motCle{here} Keyword}

The \motCle{here} keyword indicates the current parsing location is the current source file. Assigning an \nomType{@location} object from the \motCle{here} keyword is a way for initializing an \nomType{@location} object:\newline

\texttt{@location currentLocation := here ;}

\constructeurSansArgument{nowhere}
{@location}
{2.1.2}
{@location}
{Returns an \nomType{@location} that does not points out any location.}
{The returned object responds \motCle{true} to the \lienReader{@location}{isNowhere}.}


\readerSansArgument{column}
{@location}
{1.8.2}
{@uint}
{Returns an \nomType{@uint} value containing the column of the receiver's value.}
{this reader raises a run-time error if the receiver's value responds \motCle{true} to the \lienReader{@location}{isNowhere}.}


\readerSansArgument{isNowhere}
{@location}
{2.1.2}
{@bool}
{Returns an \nomType{@bool} value indicating whether the receiver'value points out a source location or does not.}
{this reader returns \motCle{true} if the receiver's value does not point out an actual location in a text source (i.e. it has been constructed using the nowhere constructor), and \motCle{false} if the receiver's value points out an actual location in a text source (i.e. it has been constructed using the \motCle{here} keyword.}


\readerSansArgument{line}
{@location}

{this reader raises a run-time error if the receiver's value responds \motCle{true} to the \lienReader{@location}{isNowhere}.}


\readerSansArgument{locationIndex}
{@location}
